# Gemini Guidelines

## Commit Messages

When creating a commit, please follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) specification.

The commit message should be in English.

### Structure

```
<type>[optional scope]: <description>

[optional body]

[optional footer(s)]
```

### Example

```
feat(login): add user authentication logic
```

## Reglas de Interacción con Git

Para asegurar una colaboración fluida y darte control total sobre el repositorio, seguiré estas reglas estrictamente:

1.  **No ejecutaré comandos de Git por iniciativa propia**: No ejecutaré `git status`, `git add`, `git commit`, ni ningún otro comando de Git a menos que tú me lo pidas explícitamente. Mi función es asistirte, no tomar el control de tu flujo de trabajo.

2.  **Proporcionar solo el mensaje de commit**: Cuando me pidas un mensaje de commit, te proporcionaré únicamente el texto del mensaje, formateado según las convenciones. No lo envolveré en un comando `git commit -m "..."` ni te sugeriré otros comandos.

3.  **Idioma de los Mensajes de Commit**: Como ya se indica en la sección de "Commit Messages", todos los mensajes de commit que yo redacte estarán siempre en **inglés**.

## Idioma de Comunicación

Para asegurar una comunicación clara y efectiva, todas mis respuestas, explicaciones, preguntas y sugerencias estarán redactadas exclusivamente en **español**.
